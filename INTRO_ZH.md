# HeptaBrain：讓 AI 在你的筆記裡漫遊

最近看到很多人分享自己的 AI 筆記工作流，從 Karpathy 的 LLM-wiki 到各種 Obsidian + AI 的整合方案。我也做了一套，但方向不太一樣。大多數人的做法是讓 AI 幫你「整理」筆記——分類、索引、摘要、歸檔。AI 當圖書館員，你當讀者。我做的東西不是整理，是「漫遊」。讓 AI 拿著你的一張卡片，在你的 Heptabase 知識庫裡亂走，撞見你自己絕對不會想到要去看的東西，然後告訴你：「欸，這兩張看起來完全無關的筆記，底層邏輯其實是同一個。」

## 語義搜尋的天花板

第一個版本的做法很直覺：用向量搜尋（semantic search），從一張卡片出發，找最相關的下一張，再找下一張。但很快就發現問題——向量搜尋找到的永遠是「同一個領域的鄰居」。從「旋轉肌袖修復」出發，它找到「十字韌帶重建」。從「CMS 綁定支付」出發，它找到「BPCI-A 模式」。這不叫跨域碰撞，這叫在同一條街上散步。

魯曼的 Zettelkasten 之所以能產出意想不到的洞見，靠的不是找到「最相關」的卡片，而是撞見「意外但相通」的卡片。語義搜尋做不到這件事。

## 升維搜尋

後來在和多個 AI（Claude、Gemini、ChatGPT）做架構審查的過程中，Gemini 指出了一個關鍵問題：向量搜尋的本質是在同一個語義平面上找最近點。要跨域，你得先離開這個平面。

方法是：在搜尋之前，先把概念「往上提一層」。不要拿「旋轉肌袖修復」去搜。先問 AI：這張卡片的底層原則是什麼？AI 可能會回答：「在機械限制下恢復功能。」然後拿「在機械限制下恢復功能」去搜。這時候你就不只會找到骨科手術，你可能會撞見「災難場域的資源分配」或「電池限制下的感測器設計」——完全不同的領域，但共享同一個底層原則。我把這個方法叫做「升維搜尋」（Dimensional Elevation）。先抽象，再搜尋。

## 第一次漫遊發生了什麼

第一次跑這套系統，我從一張關於個人知識管理的筆記出發。那張卡片的核心觀點是「觀察帶來改善」——來自 Timothy Gallwey 的《比賽，從心開始》：不要試圖控制，只要觀察，系統會自動校準。AI 把這個概念升維成「讓資訊可見化本身就會啟動自我修正迴圈」，然後用這個原則去搜。

第二步，它撞見了一張關於植入式感測器的卡片。LC 共振電路嵌入骨板中，骨折癒合時頻率會改變——把不可見的生物力學過程轉化為可觀測的電磁信號。知識管理和植入式感測器，看起來完全無關，但底層邏輯一樣：把看不見的東西變成看得見的。

第三步，它又撞見了我的災難場域作業系統。Event Sourcing 把混亂的現場操作記錄成不可篡改的事件流——把混亂變成可審計的秩序。三張卡片，三個完全不同的領域，一個底層邏輯：觀察 → 可見化 → 自我修正。

然後我突然想到一件事。這跟我讓術後病患每天回報疼痛指數，是一模一樣的道理。

我在臨床上觀察到一個現象：讓病患每天用 app 記錄「我今天有多痛」，幾乎沒有人的疼痛會維持不變或增加。不是因為記錄本身止痛，而是因為記錄讓病患「看見」了自己的下降趨勢。看見了就不怕了，不怕了就不會災難化思考，疼痛的主觀放大就消失了。回報疼痛指數本身就有止痛效果。這個臨床洞見是從一次 AI 漫遊中撞出來的。如果我自己手動翻卡片，我不可能從「知識管理」走到「LC 感測器」再走到「災難場域」然後連結回「術後疼痛追蹤」。但 AI 在 30 秒內走完了這條路。

## 實際怎麼用

操作很簡單。在 Claude Code 的 CLI 裡打一行指令：`/zettel-walk wander "任何你想探索的概念"`。AI 會在你的 Heptabase 裡走 3-5 步，每步都先升維再搜尋。走完之後，它會展示完整的路徑、發現的隱藏模式、底層邏輯、以及每條連結的證據。

然後它問你：要不要寫入今天的 Heptabase Journal？如果你說好，漫遊結果就會出現在你的 Journal 裡。你在 Heptabase 裡瀏覽 Journal，覺得好的段落右鍵「Turn into card」——一鍵就變成卡片，直接出現在你正在看的白板上。不需要手動拖卡片、不需要想位置。AI 負責發現，Journal 負責暫存，你負責決定什麼值得留下。

除了漫遊，還有三種模式。Shuffle 隨機抽幾張卡片加上你指定的一張，找共通原則。Bridge 指定兩張看似無關的卡片，AI 找橋接路徑，而且會強制找出兩者之間的張力——不只是共通點，還有衝突點。Journal 回顧最近七天的日誌，找重複出現的主題和值得升格的片段。

## 為什麼是 Heptabase

Obsidian 的生態系很強大，很多人用 Obsidian + AI 做了很厲害的東西。但 Heptabase 有一個 Obsidian 做不到的事：白板。把卡片拖到白板上、用空間位置表達概念之間的關係、一眼看到整個知識領域的結構——這是 Heptabase 的核心價值。文字連結是一維的，白板是二維的。

Heptabase 有 MCP（Model Context Protocol），讓 Claude Code 可以直接搜尋、讀取、建立卡片和寫入 Journal。目前 MCP 有限制——不能把卡片放到白板上、不能編輯或刪除——所以我設計了 Journal 中繼模式：AI 寫入 Journal，你在 Heptabase 裡決定什麼升格成卡片。這反而更符合 Zettelkasten 的精神：AI 負責發現連結，人負責判斷品質。

## 觀察本身就是干預

用了一段時間之後，我發現這套系統最有價值的不是任何單一功能，而是它背後的一個原則：觀察本身就是干預。當你讓 AI 漫遊你的知識庫，你其實是在「觀察」你自己的思考——透過 AI 的眼睛。AI 會看到你看不到的模式，因為它沒有你的認知偏見，它不會只記得最近的十張卡片。而當你看到這些隱藏的連結，你的思考就已經被改變了。不需要刻意做什麼，光是看見就夠了。

這套工作流的程式碼和設計文件我放在 GitHub 上了，有興趣的人可以直接拿去用。安裝只需要兩步：把兩個 skill 檔案複製到 Claude Code 的 commands 資料夾，然後初始化兩個 JSON registry 檔案。接下來打 `/zettel-walk wander "任何概念"` 就可以開始了。Elevation Anchors（升維錨點）記得換成你自己關注的領域，這不是本體論，只是幫助 AI 在升維時有方向感的幾條高速公路。

期待看到你的漫遊路徑。
